## 下午 会话 — TRACE 内部机制改进建议分层分析

### 做了什么
- 核实现有 attention 实际解释的是最后一层、平均头、问题末 token 对评论词的注意力，而非敏感属性目标 logit
- 将目标 Logit 归因、AttnLRP、Activation Patching、Probe/Concept Erasure、Privacy Neuron 和 A0-A3 消融整理为递进证据链
- 查阅相关论文页面并对照当前 TRACE 模型角色与配置，区分同模型白盒解释和跨家族外部评估
- 细化方案一的多 token sequence log-prob margin、Gradient×Input/IG 归因、oracle 与 predicted-target 双模式、代码落点、测试、负对照和验收指标

### 决策
- 目标 logit 用于定义“解释什么”，AttnLRP 用于低成本筛选，Top-K patching 用于因果确认，三者不是互相替代关系
- 同一白盒模型负责攻击预测与内部解释；独立跨家族 attacker/judge 只负责评估泛化与最终隐私效果
- Probe 只能说明属性可解码，Privacy Neuron 文献主要针对参数记忆，均不能直接替代输入证据定位
- 方案一先保留 raw attention 为基线，以 oracle Gradient×Input 做 shadow MVP；正式部署模式不得接收 `target_label`，多 token 属性必须按完整候选序列评分

- 尚未实现或运行 A0-A3 实验
