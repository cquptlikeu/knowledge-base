# projects/ 目录说明

## 项目目录结构

每个项目有独立的子目录：

```
projects/<project-name>/
├── context.md       ← 项目上下文（技术栈、目标、当前阻塞）
├── decisions.md     ← 架构决策记录
└── roadblocks.md    ← 踩坑记录
```

## context.md

记录项目的基本信息：技术栈、项目概述、当前状态、已完成/正在进行/阻塞的功能。

每次写项目的会话结束后，在 context.md 末尾追加简要记录——做了什么、什么决定、什么还没做完。

## decisions.md

每条决策格式：

```markdown
## YYYY-MM-DD: <决策标题>

- **背景**：为什么需要做这个决策
- **选项**：考虑过哪几个方案
- **选择**：选了哪个，为什么
- **代价**：这个选择有什么代价/风险
```

## roadblocks.md

每个坑的格式：

```markdown
## YYYY-MM-DD: <问题描述>

- **现象**：报错信息 / 异常行为
- **原因**：根因是什么
- **修复**：怎么解决的
- **预防**：以后怎么避免
```
