# Agent Loop（智能体循环）

AI Agent 的自主推理与执行循环模式：接收自然语言指令后进入「感知→规划→工具调用→观察→回答」的迭代过程，直到产生最终答案或达到最大迭代轮数。

## 核心思想

Agent 不是一次性生成答案，而是通过多轮工具调用和观察反馈逐步逼近正确答案——类似于人类面对复杂问题时的"试错-调整"过程。

## 什么时候用

- 任务需要多步骤操作（如"找出 2024 年的合同文件并统计总额"）
- 用户意图需要模型通过工具交互来理解和细化
- 结果质量比响应速度更重要

## 什么时候不要用

- 简单确定性操作（如"列出 /documents 下的所有文件"）——单次工具调用即可完成
- 对延迟极其敏感的场景
- 写操作风险高且无法承受误操作代价时——需加强确认机制

## 常见误区

- **误区**：更多迭代轮数 = 更好的结果 → 每增加一轮可能引入新的错误。本架构上限 10 轮
- **误区**：Agent 可以自由操作所有文件 → 权限应由 Permission Hook 统一控制，Agent 不自行维护权限逻辑

## 安全设计

> 写操作（移动、删除）需要二次确认，防止模型理解偏差导致误操作。
