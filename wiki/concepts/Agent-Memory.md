# Agent Memory（Agent 记忆系统）

让 AI Agent 能够跨会话持久化记忆的能力——不是每次对话都失忆的聊天机器人，而是真的记得用户说过什么、做过什么决定、有什么偏好。

## 核心思想

Agent 的价值不仅在于「当前对话中能做什么」，更在于「它能从过去的互动中学到什么」。带记忆的 Agent 能建立用户画像、追踪长期目标、避免重复提问，从「工具」进化为「助手」。

## 什么时候用

- 个人助手场景——需要记住用户的偏好、习惯、过去的决定
- 长期项目追踪——需要跨多天/多周记住项目状态和进展
- 知识积累——Agent 应该越来越了解用户，而不是每次都从头开始

## 什么时候不要用

- 一次性任务——任务结束后不需要保留任何状态
- 隐私极度敏感场景——如果记忆内容包含敏感信息，存储和检索的安全性需要额外考量
- 用户每次的需求完全独立且无关

## 三大技术挑战

| 挑战 | 说明 | 解决方向 |
|------|------|---------|
| **长期记忆存储** | 建一个能放记忆、能查记忆的存储层 | 向量数据库 + 结构化存储混合；关键记忆全量保留，细节按重要性衰减 |
| **记忆检索** | 当前需要哪些记忆？怎么从大量记忆中精准找到相关的那几条 | 语义检索 + 时间衰减 + 重要性加权 |
| **上下文压缩** | 记忆不能无限塞进上下文窗口 | 摘要化旧记忆、只保留「结论+关键事实」、按需展开详情 |

## 常见误区

- **误区**：记忆 = 把所有对话存下来 → 存储是成本最低的环节，检索和压缩才是真正的难点。无边界的记忆会让上下文膨胀，反而降低回复质量

## 相关概念对比

- **配合 [[concepts/Agent-Loop]]**：记忆系统在 Agent 的推理循环中提供历史上下文
- **配合 [[concepts/Multi-Agent-Collaboration]]**：多 Agent 系统中需要共享记忆 vs 私有记忆的边界设计
- **vs [[entities/RAG]]**：RAG 检索的是外部文档，Agent Memory 检索的是过去的交互——两者的检索技术相通，但数据来源和更新模式不同
