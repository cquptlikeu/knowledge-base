# concepts/ 目录说明

## 概念是什么

方法论、架构模式、设计理论、编程范式、思维模型。不是具体的工具，而是**怎么想的**。

## 页面结构

每个概念页面应包含以下区块：

1. **核心思想**：用一句话说清这个概念的独特洞察是什么
2. **为什么重要**：它解决了什么问题？没有它之前是什么样的？
3. **什么时候用**：适用的场景和条件
4. **什么时候不要用**：反面边界——这比适用场景更能定义一个概念
5. **常见误区**：人们常犯的错误理解或错误应用
6. **相关概念对比**：和相近但不同的概念区分开

## 示例

好概念页面不是「ABI 兼容性是二进制层面的兼容性」这种一句定义——这等于没说。好页面是：
- 解释 ABI 和 API 的区别（一个是源码契约，一个是二进制契约）
- 给具体场景（写 FFI 时需要关心，纯 Rust 内部不需要）
- 纠正误区（「大小相同≠ABI 兼容」）

## 增量更新

当新文章提到已有概念时：
- 保留所有已有内容
- 新来源中的新洞察、新场景、新误区，追加到对应区块
- 在来源列表追加新文章链接
