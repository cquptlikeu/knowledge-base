# 容器化部署

一种将应用及其依赖封装为独立镜像的部署方式。在本架构中，通过 Docker Compose 将 OpenLDAP、Samba、NFS、Nginx、Go 后端等异构服务统一编排，实现单命令部署。

## 核心思想

消除"在我机器上能运行"的环境差异问题——开发、测试、生产环境运行完全相同的镜像制品。

## 什么时候用

- 多异构服务需要协同部署（LDAP + Samba + NFS + Nginx + Go）
- 目标运维人员非专业 SRE（简化部署复杂度）
- 需要服务升级可回滚（镜像版本化）

## 什么时候不要用

- 极简系统（单进程、无外部依赖）→ 容器化的复杂度超过收益
- 对网络性能有极致要求且 NAT 开销不可接受 → 但可用 host 网络模式缓解
- 部署环境不支持容器运行时

## 特殊设计考量

采用 `network_mode: host` 而非默认 bridge 模式——SMB 的广播/多播依赖、NFS 的 RPC 端口映射要求、Samba 的 NetBIOS 设备发现在 bridge 模式下无法工作。
