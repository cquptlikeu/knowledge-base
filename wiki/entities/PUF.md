# PUF（Physically Unclonable Function）

物理不可克隆函数——利用芯片制造过程中的微观物理差异生成唯一且不可复制的密钥，为设备提供硬件级根信任锚点。

## 核心特征

- **不可克隆**：密钥源于芯片物理特征而非存储值，即使逆向工程也无法复制
- **不可伪造**：签名与硬件物理特征绑定，软件级伪造不可行
- **硬件根信任**：为存证签名提供无法伪造的硬件级私钥

## 主要应用场景

- **设备溯源**：将存证签名从软件级（可伪造）提升至硬件级（不可克隆）
- **安全启动**：固件完整性校验的硬件信任根
- **国产化合规**：CCM3302 等国密 PUF 芯片满足政务/金融合规要求

## 与其他实体的关系

- **配合 [[concepts/国产化演进]]**：PUF + SM2 签名替代软件级设备 ID 标记
- **配合 [[concepts/哈希链可信存证]]**：PUF 签名为哈希链节点提供不可伪造的溯源签名
- **配合 [[entities/SHA-256]]**：当前签名算法为 SHA-256 + 设备 ID，规划升级为 SM3 + PUF SM2

## 演进路径

架构已预留 puf-agent 微服务接口：CCM3302 PUF 芯片接入 → puf-agent 封装 PUF SDK → authd 调用签名接口 → 证明包携带 SM2 签名。
