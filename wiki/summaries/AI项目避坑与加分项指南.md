# AI 项目避坑与加分项指南

**核心论点**：2026 年应届生人人都能做 AI 项目，能拿到 offer 的从来不是你做的项目有多高大上，而是你的项目有落地、有深度、有差异化，你完全吃透了。作者分享了自己春招中踩过的 3 个巨坑和亲测面试加分的 4 个项目方向。

## 论证逻辑链

### 1. 三个巨坑（千万别做的项目）

**坑 1：纯调 API 的 demo 级项目**
基于 OpenAI API 做的通用聊天机器人，加了个简单前端页面。面试官问「除了调 API，自己做了什么核心工作？」当场哑口无言。纯调 API 的 demo 在 2026 年毫无竞争力。

**坑 2：烂大街的基础 RAG 知识库**
LangChain + Chroma + GPT 做本地文件知识库问答。面试官：「这个项目我们面的人基本都做过，你跟别人的区别在哪？」基础 RAG 已经是 2024 年的应届生项目，2026 年已经烂大街。

> 面了 5 家，4 家面试官都跟我说：「这个项目我们面的人基本都做过，你跟别人的区别在哪？」

**坑 3：盲目做太硬核的算法项目但自己 hold 不住**
跟风做 Llama 3 微调，面试官追问底层原理、数据处理、优化细节时答不上来。面试官评价：「看起来很硬核，但你自己都没吃透，还不如做一个小而精的。」

### 2. 亲测面试加分的 4 个 AI 项目

| # | 项目 | 适配岗位 | 核心技术与亮点 |
|---|------|---------|---------------|
| 1 | **基于 MCP 协议的电商运营垂直 Agent** | AI 应用开发、Java 后端、全栈 | MCP 协议对接淘宝/京东/飞书/Excel/千牛；自动订单分析、差评监控、商品标题优化、客服话术生成、投放数据复盘；运营复盘时间从 2h 压到 10min，标题优化后 CTR +18% |
| 2 | **工业级多模态 RAG 产品质检系统** | AI 应用开发、后端、算法 | 多模态 RAG 融合图片+文档+视频；从基础 RAG 升级为有场景落地的系统；问答准确率从 62% 优化到 94%，完整优化链路 |
| 3 | **基于本地大模型的代码审计 Agent** | Java 后端、Go 后端、AI 应用、安全 | Llama 3 8B 本地部署；对接 Git+Jenkins；安全漏洞/性能/规范扫描+修复建议+自动生成修复代码；漏洞检出率 70%→88%，误报率<15% |
| 4 | **端侧轻量化 AI 日程管理 Agent** | 客户端、前端、AI 应用 | Qwen2 1.5B 端侧运行；读取短信/微信/日历自动生成日程；模型量化+推理优化，速度×3，内存-60%；不上传数据保护隐私 |

### 3. 四个项目的共同成功要素

1. **垂直场景落地**：不是通用 Agent，是解决某个行业/场景的真实痛点
2. **踩中技术风口**：MCP 协议（2026 年最火）、端侧 AI、多模态 RAG
3. **工程化落地**：不是单文件 demo，有完整的权限管理、任务调度、日志监控
4. **真实优化数据**：每个项目都有可量化的优化结果（准确率、时间、检出率、CTR）

> 花 1 个月时间，做一个垂直场景、小而精、有真实优化的 AI 项目，比你做 10 个 demo 都管用。
