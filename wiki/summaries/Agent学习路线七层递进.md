# Agent 学习路线七层递进（后端转 Agent 版）

**核心主张**：2026 年后端卷到没边，AI Agent 岗位占主导。作者给出了一套「最少必要知识路线」——不堆概念、不啃论文，只学面试必问、项目必用、能快速写进简历的硬核内容，3 个月从 0 到能投 Agent 岗。

## 定位

这套路线特别针对 **Java 后端开发者转 Agent**。核心洞察：**后端服务 = Agent 的执行器**——Agent 做决策，你写逻辑。后端的 API 设计、鉴权、线程池、限流熔断、MySQL/Redis、SpringBoot 校验异常这些底子，不需要重学，只需要对齐到 Agent 体系。

> Agent = LLM 大脑 + 工具调用 + 记忆 + 规划调度

## 论证逻辑链

### 第一阶段：Agent 核心基础

1. **LLM 大模型基础**（面试必问）：能力边界、提示词工程（Zero-shot/Few-shot/CoT/结构化输出）、国产模型接口调用
2. **Agent 核心四组件**：感知（理解意图）→ 记忆（短期/长期/向量）→ 规划（拆任务、做决策）→ 执行（调工具、调 API、写库表）
3. **ReAct 范式**：Reasoning → Acting → Observation → 循环。面试常考题：为什么 ReAct 比单纯 LLM 更稳？
4. **工具调用 Function Calling**：函数定义、注册、路由、异常捕获，实战调天气/数据库/Redis/支付接口

### 第二阶段：Agent 开发框架核心

1. **LangChain**（必学）：Chain、PromptTemplate、OutputParser、工具封装、AgentExecutor、记忆组件。不必全看完，按需跳着学。
2. **LangGraph**（进阶必学）：状态节点、条件跳转、循环、中断恢复，多步骤任务。对比 LangChain vs LangGraph。
3. **多 Agent 框架**：CrewAI（角色分工、任务调度）、AutoGen（多轮对话、人机协同）

### 第三阶段：Agent 关键技术——RAG 为什么是必学

作者花了大量篇幅论证 RAG 对后端转 Agent 的特殊意义：

**两个致命缺陷**迫使纯 LLM 的 Agent 只能停留在 Demo 阶段：
1. 训练数据固定且滞后——企业内部文档、实时业务数据未进入训练集，模型无感知
2. 幻觉问题——模型会编造信息，放到企业场景引发合规风险

**RAG 是最优解**：不需要微调模型，不改模型权重，只通过外部知识库检索信息 → 作为上下文传给大模型 → 模型基于事实生成回答。

> 对于后端开发者来说，RAG 本质上就是把我们熟悉的数据库查询、搜索引擎检索、缓存优化等技术，和大模型生成能力结合起来——学习成本远低于其他 AI 技术。

**RAG 五步闭环与后端技术栈的对应**：

| RAG 步骤 | 后端对应 |
|-----------|---------|
| 文档加载与预处理（PDF/Word/Excel/数据库） | 数据导入、文件处理、格式清洗 |
| 文本切片（chunk 大小 + overlap） | 数据分片、分页处理 |
| 向量化嵌入（调开源 Embedding 模型接口） | 不需要自己训练，调接口即可 |
| 检索召回（向量相似度 + 关键词混合检索 + 重排） | 索引查询、精准检索、混合检索 |
| 提示词拼接与生成 | 调大模型 API |

### 第四阶段：项目实战

**项目 1 — 企业级智能客服 Agent**：SpringBoot + LangChain + Redis + MySQL + RAG，意图识别/多轮对话/订单查询/退款流程/工具调用/会话记忆/限流熔断。

**项目 2 — 多 Agent 协同研报助手**：LangGraph + CrewAI + 向量库 + 文档解析，采集→分析→写作→审核四角色协作。

### 关键提醒：Demo ≠ 企业级

> 企业级 RAG 系统从来不是简单的文档切片加向量检索再加模型生成。Demo 只用几份干净文档测试；企业要面对 PDF 扫描件、复杂表格、加密文件、多格式混合内容、增量同步、权限控制、高并发、全链路监控。

**企业级 RAG 必须具备的工程能力**：
- 数据层：格式兼容、噪声过滤、敏感信息脱敏、版本管理、细粒度权限
- 性能层：Redis 缓存、向量索引优化、上下文窗口控制、线程池+异步+限流熔断
- 质量层：混合检索+语义重排+多路召回、兜底机制、全链路日志监控
- 架构层：模型/向量库/Embedding 的灵活切换，预留多 Agent 协同扩展空间
